# config-weave — processes

Each process is a runbook on its own page. This is the index.

- [**Scaffold and validate a playbook**](../references/process_scaffold_validate.md) — Start a new playbook from the built-in skeleton and confirm it validates before running anything.

- [**Add a package resource**](../references/process_add_resource.md) — Declare a new resource in a package and implement its check/apply wscript script against the host API.

- [**Test a package for idempotence**](../references/process_test_package.md) — Prove a package's resources converge and stay converged using the testlab's three-run protocol in a disposable instance.

- [**Check, then apply a play**](../references/process_check_then_apply.md) — Preview what a play would change (report-only), then converge the machine.
